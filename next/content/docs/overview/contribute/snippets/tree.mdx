To display the structure of a directory with files and collapsible subdirectories, use the `<Tree>` component with `<Tree.Folder>` and `<Tree.File>` sub-components. Click on a folder to open or close it.

Since `<Tree>` uses JSX children, you can use standard MDX content inside — links, bold text, lists, and other Markdown formatting work natively.

## Usage

No imports are needed — the component is available globally.

```mdx
<Tree>
  <Tree.File name="README.md" />
  <Tree.Folder name="src" defaultOpen>
    <Tree.File name="index.ts" />
    <Tree.File name="utils.ts" />
  </Tree.Folder>
  <Tree.File name="package.json" />
</Tree>
```

### File descriptions

`<Tree.File>` accepts children, which render as a description below the filename.

```mdx
<Tree>
  <Tree.Folder name="contracts" defaultOpen>
    <Tree.File name="first_contract.tolk">Main contract file</Tree.File>
  </Tree.Folder>
  <Tree.Folder name="wrappers" defaultOpen>
    <Tree.File name="FirstContract.ts">Wrapper class for the smart contract</Tree.File>
    <Tree.File name="FirstContract.compile.ts">Configuration for compiling contract</Tree.File>
  </Tree.Folder>
</Tree>
```

### Folder descriptions

`<Tree.Folder>` renders all children inside the folder when it is open. Besides nested `<Tree.File>` and `<Tree.Folder>`, you can place any MDX content — text, links, lists, bold, etc.

```mdx
<Tree>
  <Tree.Folder name="contracts" defaultOpen>
    Smart contract source code and imports.
  </Tree.Folder>
  <Tree.Folder name="wrappers" defaultOpen>
    TypeScript interface classes for all contracts **except Tact**.

    Each wrapper implements the `Contract` interface from [`@ton/core`](https://www.npmjs.com/package/@ton/core).

    - Provides message serialization and deserialization, getter wrappers, and compilation helpers.
    - Used in both the test suite and client code to interact with contracts from TypeScript.
  </Tree.Folder>
  <Tree.Folder name="build" defaultOpen>
    Compilation artifacts generated by the build command.
  </Tree.Folder>
</Tree>
```

### Nested folders

```mdx
<Tree>
  <Tree.Folder name="app" defaultOpen>
    <Tree.File name="layout.tsx" />
    <Tree.File name="page.tsx" />
    <Tree.Folder name="api" defaultOpen>
      <Tree.Folder name="auth">
        <Tree.File name="route.ts" />
      </Tree.Folder>
      <Tree.File name="route.ts" />
    </Tree.Folder>
    <Tree.Folder name="components">
      <Tree.File name="button.tsx" />
      <Tree.File name="dialog.tsx" />
      <Tree.File name="tabs.tsx" />
    </Tree.Folder>
  </Tree.Folder>
  <Tree.File name="package.json" />
</Tree>
```

## `<Tree.File>` props

| Prop       | Type        | Description                         |
| ---------- | ----------- | ----------------------------------- |
| `name`     | `string`    | Filename displayed in the tree      |
| `icon`     | `ReactNode` | Custom icon (replaces default icon) |
| `children` | `ReactNode` | Description rendered below the name |

## `<Tree.Folder>` props

| Prop          | Type        | Description                                                        |
| ------------- | ----------- | ------------------------------------------------------------------ |
| `name`        | `string`    | Folder name displayed in the tree                                  |
| `defaultOpen` | `boolean`   | Whether the folder is open on load, default `false`                |
| `children`    | `ReactNode` | Nested tree items and/or MDX content rendered as folder description |
