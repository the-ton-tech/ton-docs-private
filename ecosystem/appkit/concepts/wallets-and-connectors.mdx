## Wallet

The user's signing application. It holds the keys and signs transactions and data when the user approves.

AppKit collects wallets from every connector into one list. Multiple connected wallets are supported. One wallet is marked as **selected**, and AppKit uses that wallet for transfer and signing actions.

Every wallet exposes the same minimum interface: a reference to the connector that produced it, four identity readers (`getAddress`, `getPublicKey`, `getNetwork`, `getWalletId`), and two wallet-approved operations (`sendTransaction`, `signData`) that are routed through the connector.

```ts
interface WalletInterface {
  /** Connector that created this wallet */
  readonly connectorId: string;

  // Identity
  getAddress(): UserFriendlyAddress;
  getPublicKey(): Hex;
  getNetwork(): Network;
  getWalletId(): string;

  // Actions requiring wallet signature
  sendTransaction(request: TransactionRequest): Promise<SendTransactionResponse>;
  signData(payload: SignDataRequest): Promise<SignDataResponse>;
}
```

Wallet ids are derived from `(network, address)`. The same address on a different network produces a different wallet object — apps that move users between mainnet and testnet should not assume id stability across that boundary.

## Connector

A connector is the transport between AppKit and a wallet. It opens the pairing flow, reports connected wallets, and routes signing requests to the wallet.

A connector is not a wallet and does not sign. Signing always happens inside the user's wallet application; the connector only carries requests to it and responses back.

A connector implements the `Connector` interface. Connector ids are unique — registering a second connector with the same id calls `destroy()` on the old one and replaces it.

```ts
interface Connector {
  readonly id: string;
  readonly type: string;
  readonly metadata: ConnectorMetadata;
  destroy(): void;
  connectWallet(network?: Network): Promise<void>;
  disconnectWallet(): Promise<void>;
  getConnectedWallets(): WalletInterface[];
}
```

The bundled `createTonConnectConnector` covers most apps. It wraps `@tonconnect/ui`, restores prior sessions when the connector is first read, and syncs TonConnect with AppKit's default network.

## Adding a connector

Both paths accept the same input: a `Connector` instance or a factory `(ctx) => Connector`. Both use the same internal registration flow, so choose the path based on when the connector configuration is available.

### At construction

Pass connectors in the `connectors` field of `AppKitConfig`. The constructor runs `addConnector` for each entry in order. See [Get Started > Connectors](/ecosystem/appkit/get-started/connectors) for the constructor walkthrough.

### After construction

The `addConnector` action registers a connector on an existing `AppKit` instance and returns an unregister function. Use it when the connector depends on values that are not available at construction time, such as a user-scoped manifest URL, or when the app needs to replace connectors dynamically. See [How to > Connect to a wallet → Register a connector after construction](/ecosystem/appkit/howto/connect-to-a-wallet#register-a-connector-after-construction) for the code.

## React hooks

`@ton/appkit-react` exposes hooks for the main wallet and connector actions and state readers: `useConnect`, `useDisconnect`, `useConnectors`, `useConnectedWallets`, `useSelectedWallet`, and `useAddress`. See [How to > Connect to a wallet](/ecosystem/appkit/howto/connect-to-a-wallet) for the full picker and disconnect-button patterns.

Mutation hooks return TanStack Query's `{ mutate, mutateAsync, isPending, error }`. Async data hooks such as `useBalance` return TanStack Query results. Wallet and connector state hooks read AppKit's local store directly: `useSelectedWallet()` returns `[wallet, setWalletId]`, `useConnectedWallets()` returns `WalletInterface[]`, `useConnectors()` returns `Connector[]`, and `useConnectorById(id)` returns `Connector | undefined`. Watch hooks such as `useWatchBalance` return `void` and keep the related query cache fresh while mounted.

For the complete hook surface, see the [`@ton/appkit-react` reference](/ecosystem/appkit/reference/appkit-react). For the underlying core actions, see the [`@ton/appkit` reference](/ecosystem/appkit/reference/appkit).

`setSelectedWalletId` updates the stored id but does not emit `wallets:selection-changed`. The selection event fires only when the connected wallet list changes — through `connect`, `disconnect`, `addConnector`, or the auto-select that runs when the previously selected wallet drops off. After calling `setSelectedWalletId` directly, call `refetch()` on any query hooks that depend on the selected wallet.

If the `connectorId` you pass to `connect` is not registered, the action throws `Connector with id "<id>" not found`. Call `getConnectors` (or `useConnectors` in React) to confirm what's registered before invoking `connect`.

## Rules

- Do not call mutations or signing actions while `getSelectedWallet` is `null`. The action will throw.
- Do not assume every connector supports every flow. Catch rejections from the wallet and surface a recovery option.
- Do not store `selectedWalletId` as long-lived application state. The selected wallet can change after disconnects, network changes, or session restores.
- In application code, use public actions or React hooks instead of calling connector methods directly, so AppKit's event flow stays consistent. Connector methods are still the public surface for custom connector implementations.
