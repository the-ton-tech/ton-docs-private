[Jettons](/standard/tokens/jettons/overview) are fungible tokens on TON, similar to ERC-20 tokens on Ethereum. Unlike Toncoin, which is the native TON currency used in all transfers, each jetton has a separate master (minter) contract and an individual wallet contract for each holder.

For example, USDT on TON is implemented as a jetton, and its minter contract address is `EQCxE6mUtQJKFnGfaROTKOt1lZbDiiX1kCixRv7Nw2Id_sDs`. By providing this address and the recipient's TON wallet contract address, AppKit knows which tokens to send and to whom.

<Note>
  To deploy and mint a new jetton on the mainnet without writing code, use the dedicated official tool: [TON MINTER](https://minter.ton.org).
</Note>

## Jetton wallet contract

Each jetton holder has a dedicated jetton wallet contract derived from `(jettonMaster, ownerAddress)`. The owner is the user's TON wallet; the jetton wallet is a separate contract owned by it. Transfers always read the destination **owner address** — AppKit derives the recipient's jetton wallet internally.

If the recipient has never held this jetton, the transfer deploys the recipient's jetton wallet contract on first receive, which costs additional gas. The default outer-TON allowance on a built `transferJetton` request is designed to cover this case.

See [How to > Read balances → Jetton wallet address](/ecosystem/appkit/howto/read-balances#jetton-wallet-address) for the resolver code.

## Metadata and decimals

Each jetton stores `name`, `symbol`, and `decimals` in its metadata. The `decimals` field is critical for transfers and display — balance units vary across jettons. USDT uses 6 decimals, so a fractional balance string of `'0.1'` represents 100000 micro-USDT.

Read decimals from chain metadata before building a transfer. Different jettons can use different precision.

See [How to > Send jettons → Look up jetton metadata](/ecosystem/appkit/howto/send-jettons#look-up-jetton-metadata) for the read code.

## Balance

Like Toncoin, a discrete one-off jetton balance check has limited value on its own — use it only when assembling an outgoing transfer. For UI display, use continuous monitoring through the streaming layer.

See [How to > Read balances](/ecosystem/appkit/howto/read-balances) for single-jetton and all-jettons read code, and [Get Started > Streaming](/ecosystem/appkit/get-started/streaming) for live updates.

## Transfers

<Warning>
  **Funds at risk.** Each jetton stores the `decimals` parameter in its [metadata](#metadata-and-decimals). Transferring without accounting for decimals can result in sending drastically more or fewer tokens than intended.

  Mitigation: Verify the correct `decimals` value before calculating transfer amounts. For USDT, decimals is 6.
</Warning>

Before making a transfer, make sure there is enough Toncoin on the sender wallet to cover the [fees](/foundations/fees).

A jetton transfer carries the destination owner address, the jetton master address, the amount (in fractional jetton units), the decimals, and an optional comment. AppKit derives the sender's jetton wallet, builds the inner jetton transfer payload, and wraps it in an outer TON message.

See [How to > Send jettons](/ecosystem/appkit/howto/send-jettons) for the component, hook, and core action.

## Settlement

Jetton transfers settle through the same trace API as Toncoin transfers. A jetton transfer usually produces multiple internal messages; use `onchainMessages`, `pendingMessages`, and `totalMessages` on the `getTransactionStatus` response to track progress. Treat only `status === 'completed'` as final.

## Rules

- Always pass the destination **owner address** as `recipientAddress`. AppKit derives the sender's jetton wallet contract and writes the destination owner address into the jetton transfer payload.
- Read decimals from chain metadata (`getJettonInfo`) before building a transfer. Different jettons can use different precision.
- Treat the `boc` from `sendTransaction` as "user signed". Verify settlement with `getTransactionStatus` before crediting an order.
- Refetch the user's jetton list and balances after settlement.
- Do not display jetton metadata without sanitizing — names and images are user-supplied.
