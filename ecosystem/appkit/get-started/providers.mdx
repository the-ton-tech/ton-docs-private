Providers connect AppKit to external services: chain reads, live updates, swap transactions, and staking transactions. They sit on the AppKit instance and supply data; they do not own UI, sessions, or signing. See [Providers concept](/ecosystem/appkit/concepts/providers) for the model.

## Before you begin

You need AppKit packages installed and an `AppKit` instance created. See [Installation](/ecosystem/appkit/get-started/installation) and [Connectors](/ecosystem/appkit/get-started/connectors). The React `AppKitProvider` and `QueryClientProvider` are set up in [Installation → Set up React providers](/ecosystem/appkit/get-started/installation/react-app#set-up-react-providers); this page is only about data and DeFi providers.

## Provider types

Provider types attach at different points. API clients live in `networks[chainId].apiClient`. Swap and staking providers go in `AppKitConfig.providers`. Streaming providers are registered after construction on `appKit.streamingManager`; watch hooks and `watch*` actions are no-ops until one is registered.

| Type               | Where it goes                                                             |
| ------------------ | ------------------------------------------------------------------------- |
| API client         | `networks[chainId].apiClient` — `ApiClientToncenter` or `ApiClientTonApi`. |
| Streaming provider | `appKit.streamingManager.registerProvider(...)` after construction.       |
| Swap provider      | `AppKitConfig.providers` array — `DeDustSwapProvider`, `OmnistonSwapProvider`. |
| Staking provider   | `AppKitConfig.providers` array — `createTonstakersProvider({})`.          |

## Add a swap provider

Swap providers ship as separate entrypoints under `@ton/appkit/swap/*` so the bundler only includes the protocols you actually use. Construct an instance and pass it in the `providers` array on the `AppKit` constructor; AppKit registers it with the swap manager and the swap hooks (`useSwapQuote`, `useBuildSwapTransaction`) start returning quotes from that protocol.

```ts
// appkit.ts
import { AppKit } from '@ton/appkit';
import { DeDustSwapProvider } from '@ton/appkit/swap/dedust';

export const appKit = new AppKit({
  // networks, connectors...
  providers: [
    new DeDustSwapProvider(),
  ],
});
```

You can pass more than one swap provider in the same array — for example `new DeDustSwapProvider()` and `new OmnistonSwapProvider()` together — and the swap manager picks the better quote per request. Use `createTonstakersProvider({})` from `@ton/appkit/staking/tonstakers` the same way for staking.

Omniston requires its own SDK package — install it before registering the provider:

```bash
npm i @ston-fi/omniston-sdk
```

## Rule

Provider responses are external data. Validate critical outcomes server side before changing product state — wallet acceptance is not settlement, and a swap quote is not a price guarantee.
