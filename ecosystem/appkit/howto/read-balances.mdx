Read Toncoin and jetton balances for the connected wallet or any address.

## Before you begin

You need a connected wallet or known address, and the React provider must be mounted. See [Connect to a wallet](/ecosystem/appkit/howto/connect-to-a-wallet).

## Pick the right hook

| Goal                                           | Hook                  |
| ---------------------------------------------- | --------------------- |
| Toncoin balance for the selected wallet        | `useBalance`          |
| Toncoin balance for an explicit address        | `useBalanceByAddress` |
| Jetton balances for the selected wallet        | `useJettons`          |
| Jetton balances for an explicit address        | `useJettonsByAddress` |

All four are query hooks. They return `{ data, isLoading, error, refetch }` and accept a `query` field to forward TanStack Query options such as `refetchInterval`.

## Read the connected wallet

```tsx
import { useBalance, useJettons } from '@ton/appkit-react';

export function Balances() {
  const ton = useBalance({ query: { refetchInterval: 20000 } });
  const jettons = useJettons({ query: { refetchInterval: 20000 } });

  if (ton.isLoading || jettons.isLoading) return <span>LoadingŌĆ”</span>;
  if (ton.isError) return <span>Failed to load TON</span>;

  return (
    <section>
      <p>TON: {ton.data ?? '0'}</p>
      <p>Jettons: {jettons.data?.jettons.length ?? 0}</p>
    </section>
  );
}
```

`ton.data` is a fractional-Toncoin string (already formatted ŌĆö e.g. `'1.25'` for 1.25 TON), or `null` if no wallet is selected. No `fromNano` conversion needed for display.

## Read by address

Use the `*ByAddress` hooks to read any address ŌĆö useful for receipt screens or admin tooling.

```tsx
import { useBalanceByAddress, useJettonsByAddress } from '@ton/appkit-react';

export function ProfileBalances({ address }: { address: string }) {
  const ton = useBalanceByAddress({ address });
  const jettons = useJettonsByAddress({ address });
  return <p>TON: {ton.data ?? '0'}</p>;
}
```

## Render a jetton row

The SDK ships `getFormattedJettonInfo` to normalize jetton metadata for display.

```tsx
import { getFormattedJettonInfo } from '@ton/appkit';

const formatted = getFormattedJettonInfo(jetton);
// formatted: { name, symbol, decimals, balance, image, address }
```

## Jetton wallet address

Each jetton holder has a dedicated jetton wallet contract derived from `(jettonMaster, ownerAddress)`. Resolve it explicitly only when you need to display that address. `transferJetton` derives the recipient's jetton wallet internally.

```tsx
import { useJettonWalletAddress, useAddress } from '@ton/appkit-react';

const ownerAddress = useAddress();
const { data: walletAddress, isLoading } = useJettonWalletAddress({
  ownerAddress: ownerAddress ?? '<TON_WALLET_ADDRESS>',
  jettonAddress: '<JETTON_MINTER_ADDRESS>',
});
```

From vanilla JS:

```ts
import { getJettonWalletAddress, getSelectedWallet } from '@ton/appkit';

const wallet = getSelectedWallet(appKit);
const address = await getJettonWalletAddress(appKit, {
  jettonAddress: '<JETTON_MINTER_ADDRESS>',
  ownerAddress: wallet?.getAddress() ?? '<TON_WALLET_ADDRESS>',
});
```

## Read from vanilla JS

```ts
import { getBalanceByAddress, getJettonsByAddress } from '@ton/appkit';

const ton = await getBalanceByAddress(appKit, { address: 'EQ...' });
const jettons = await getJettonsByAddress(appKit, { address: 'EQ...' });
```
